应用密码学与网络安全（一）

发表于 2022-01-12 更新于 2022-03-14 分类于 应用密码学与网络安全 阅读次数： Valine：

应用密码学与网络安全的引言与概述。

引言

如何保证 Web 服务器和 Web 服务器之间的安全”交流“？

Ans： 使用 HTTPS 协议。

SSL 协议

SSL 安全套接层协议，专门用于保护 web 通信。

SSL 协议通过加密传输来确保数据的机密性，通过消息验证码机制来保护信息的完整性，通过数字证书来对发送方和接收方的身份进行认证。

三个关键概念

  • 计算机安全（Computer Security）

​ 对于一个自动化的信息系统，采取保护措施确保信息系统（包括硬件、软件、固件、信息/数据和通信）的保密性、完整性、可用性。

​ 是针对一个计算机自身的安全。

  • 网络安全

​ 保护数据在网络传输中安全的方法。

  • 互联网安全

​ 保护数据安全通过互联网络的方法。

网络安全核心需求

  • 保密性：数据保密性、隐私性。
  • 完整性：数据完整性、系统完整性。
  • 可用性
  • 真实性
  • 可追溯性

概述

网络安全现状

Internet 一方面成为人们离不开的信息工具，同时也成为公开的攻击对象目标。网络的全球性、开放性、无缝连通性、共享性、动态性，使任何人都可以自由地接入 Internet，其中有善者也有恶者。恶意者时刻在试图穿透别人的系统，捣毁别人的信箱、散步破坏性信息、倾泻信息垃圾。

OSI 安全框架

  • 安全攻击 security attack 问题

任何危及信息安全的活动。分为主动攻击和被动攻击。

被动攻击是在未经用户同意和认可的情况下将信息或数据文件泄露给系统攻击者，但不对数据信息做任何修改。常见手段有搭线监听、无线截获、其他截获等，不易被发现。防范被动攻击的重点在于预防，如使用虚拟专用网（VPN）、采用加密技术保护网络以及使用加保护的分布式网络等。

主动攻击时，攻击者会对消息做处理后再传给接收者，涉及某些数据流的篡改或虚假流的产生。通常分为假冒、重放、篡改消息、拒绝服务等。主动攻击能够检测出来，但不易有效防止，具体措施包括自动审计、入侵检测和完整性恢复等。

拒绝服务攻击（DoS），是通过大量虚假访问耗尽被攻击对象的资源，让目标计算机或网络无法提供正常的服务或资源访问，使目标系统服务停止响应甚至崩溃。

分布式拒绝服务攻击（DDoS），是指黑客利用并控制已经侵入的主机进行协同 DoS 攻击，比一般 DoS 攻击更自动化、更隐蔽且更难防范。

  • 安全服务 security service 效果

加强数据处理系统和信息传输的安全性的一种服务。目的在于利用一种或多种安全机制阻止安全攻击，对文档进行安全地分发，是对安全攻击的反击。

设计安全服务应包含四个方面内容：

  1. 设计执行安全相关传输的算法，
  2. 产生算法所使用的秘密信息，
  3. 设计分配和共享秘密信息的方法，
  4. 指明通信双方使用的协议，该协议利用安全算法和秘密信息实现安全服务。
  • 安全机制 security mechanism 方法

用来保护系统免受侦听、阻止安全攻击及恢复系统的机制。

安全机制要具备检测、防御或从安全攻击中恢复的能力，没有单一的机制能够提供所有的安全服务，一个机制往往构成其他安全机制的基础，比如加密技术。

特定的安全机制有加密、数字签名、访问控制、数据完整性、认证交换、流量填充、路由控制、公证等。

普遍的安全机制有可信功能、安全标签、事件检测、安全审计跟踪、安全恢复等。

信息加密分为对称密钥体制和非对称密钥体制（采用公钥）。

对称密钥体制的加密和解密采用同一个秘密密钥，通信双方在安全的秘密信道协商好密钥后，发送方将信息明文通过加密算法和密钥加密成密文，密文沿公开信道传输给接收方，接收方收到密文后使用解密算法和密钥解密信息明文。

非对称密钥体制，发送方用接收方的公钥加密信息明文，接收方用自己的私钥对密文解密。公钥和加解密算法是公开的，只有用于解密的私钥是保密的。非对称密码体制利用了计算复杂性原理。

消息摘要保护信息的完整性，发送方将信息明文通过 MAC 算法进行摘要，然后用自己的私钥对摘要进行加密（数字签名），将加密后的摘要附加到信息明文尾部，然后将信息发生给接收方。接收方收到信息后，将信息内容通过 MAC 算法进行摘要，将发送方传来的摘要密文用发送方的公钥解密，将解密后的摘要与 MAC 算法得到的摘要进行比对，就能知道信息的完整性是否被破坏。

数字签名是传统签名的数字化，能与所签文件”绑定“，签名者不能否认自己的签名，签名不能被伪造，且容易被自动验证。数字签名机制基于签名过程和验证过程，签名过程是利用签名者的私有信息作为私钥，对数据单元进行加密或产生该数据单元的密码校验值，验证过程是利用公开的规程和信息来确定签名是否是利用该签名者的私有信息产生的。

  • 安全协议

安全协议是建立在密码体制基础上的一种高互通协议，它运行在计算机通信网或分布式系统中，为安全需求的各方提供一系列保障。借助密码算法来达到密码分配、身份认证、信息保密等目的。

密钥交换协议：完成会话密钥的建立与认证协议结合使用。

认证协议：实体认证、消息认证、数据源认证等，用于防止假冒、篡改、否认等攻击。

认证密钥交换协议：对通信实体的身份认证，成功认证后，为下一步的安全通信分配密钥。

电子商务协议：保证交易双方都不能通过损害对方利益而得到它不应得到的利益。

网络安全访问模型

需要做到选择合适的门卫功能以识别用户，实现安全控制以确保只有授权用户才可以访问被指定的信息或资源。

计算机安全

病毒

病毒是一种特殊的计算机程序，基于硬件和操作系统，会进行一些恶意活动破坏用户的网络或信息系统。任何一种病毒都是针对某种处理器和操作系统编写的。

计算机病毒的特点：破坏性、传染性、隐藏性、可激活性、针对性。

病毒传播的主要途径：网络、U盘、硬盘和光盘。

计算机病毒的类型：

系统引导型病毒：寄生在磁盘引导区或主引导区。

文件型病毒：寄生在可执行程序。

宏病毒：Word 宏病毒是利用 Word 提供的功能，将病毒程序插入到带有宏的 .doc 文件或.dot 文件中。

混合型病毒：同时具有多种病毒的特征，破坏性大，传染机会多，灭杀困难。

网络蠕虫病毒：通过网络传播的恶性病毒，具有自己的一些特征，如不利用文件寄生（如只存在于内存），使服务器拒绝服务或和其他黑客技术相结合使用等。

木马病毒：伪装潜伏，常通过一段特定的程序（木马程序）来控制远程计算机。

计算机病毒防护：

查毒引擎：无界面和包装的核心程序，在杀毒软件所安装的目录下。

杀毒软件：病毒特征码、查毒引擎、精美的操作画面。

病毒扫描技术：已知病毒扫描（特征码扫描技术、广谱特征扫描技术）、未知病毒扫描（虚拟机技术）。

访问控制机制

访问控制的目的是防止对信息资源的非授权访问和使用。用来实施对资源访问或操作加以限制的策略。保证网络资源不被非法使用和访问。在身份认证后实施。

防火墙

传统防火墙被设计用来防止火从大厦的一部分传播到另一部分。

外部网络被认为是不安全和不可信赖的，内部网络被认为是安全和可信赖的，防火墙一般安放在被保护网络的边界。

防火墙是位于两个或多个网络之间，执行访问控制策略的一个或一组系统，是一类防范措施的总称。它是不同网络安全域间通信流的唯一通道，能根据企业有关的安全政策控制（允许、拒绝、监视、记录）进出网络的访问行为。

入侵检测系统

入侵检测系统是为保证计算机系统的安全而设计与配置的一种能够及时发现并报告系统中未经授权的访问和其他异常现象的技术，是一种用于检测计算机网络中违反安全策略行为的技术。

入侵检测被认为是防火墙之后的第二道安全闸门，在不影响网络性能的情况下能对网络进行监测，从而提供对内部攻击、外部攻击和误操作的实时保护。

入侵检测系统的功能是发现可能存在的安全攻击。

入侵检测系统的工作流程：信息收集、数据分析、响应。

检测方法：

  1. 基于攻击特征行为：根据入侵行为的特征建立入侵行为模型，若当前行为过程与某个入侵行为特征一致，判定入侵发生。适合检测已知攻击。
  2. 误用检测：建立正常网络活动的特征模型，若当前行为过程与正常模型不符，判定入侵发生。适合检测未知攻击。

入侵检测方法的评估：检测率、误报率。
