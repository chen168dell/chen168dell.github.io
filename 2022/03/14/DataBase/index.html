2021期末数据库复习总结

发表于 2022-03-14 分类于 数据库系统概论 阅读次数： Valine：

因为疫情导致的第四轮复习的产物，数据库真让人麻木 orz。

主要是一些概念。和代码相关的还有关系代数关系演算部分就不搞了，看看书就会。

一、绪论

数据

描述事物的符号记录。

数据的含义称为数据的语义，数据与其语义是不可分的。

数据库

长期存储在计算机内，有组织，可共享的大量数据的集合。

特点：较小的冗余度、较高的数据独立性和易扩展性。

数据库管理系统

位于用户应用与操作系统间的一层数据管理软件。

数据库系统

由数据库、数据库管理系统（及其应用开发工具）、应用程序和数据库管理员组成的存储、管理、处理和维护数据的系统。

数据库系统特点

  • 数据结构化。
  • 数据共享性高、冗余度低且易扩充。
  • 数据独立性高。
  • 数据由数据库管理系统统一管理和控制。

数据模型

概念模型（信息模型）

逻辑模型：按计算机系统的观点对数据建模，用于 DBMS 实现。

物理模型：描述数据在系统内（磁盘上）的表示形式和存取方式。

一些概念

  • 实体：人、书、物。
  • 属性：特性。
  • 码（key）：属性集。
  • 实体型：用实体名及其属性名集抽象和刻画同类实体。
  • 实体集：同类型实体集合。
  • 联系：1对1、1对n、m对n （E-R图）。

数据模型的组成要素

  • 数据结构：描述数据库的组成对象以及对象之间的联系。
  • 数据操作：查询和更新（包括插入、删除、修改）。
  • 数据完整性的约束条件：一组完整性规则。

数据独立性

  • 物理独立性：用户的应用程序与数据库中的数据的物理存储相互独立。
  • 逻辑独立性：用户的应用程序与数据库的逻辑结构相互独立。

两代数据模型

第一代数据模型

层次模型，树形结构。

网状模型，网状结构。

第二代数据模型

关系模型。

关系模型

  • 关系名：表名。
  • 关系模式：表头。
  • 关系：二维表。
  • 元组：记录或行。
  • 属性：列。
  • 属性名：列名。
  • 属性值：列值。
  • 分值：一条记录中的一个列值。
  • 非规范关系：表中有表。

关系的完整性约束条件

实体完整性、参照完整性、用户定义完整性。

实体完整性和参照完整性是关系的两个不变性。

三级模式结构

外模式：数据的逻辑独立性。用户能见的局部数据的逻辑结构和特征描述。

模式：数据库中全体数据的逻辑结构和特征描述。

内模式：数据的物理独立性

外模式可以有多个，模式和内模式只有一个。

二、关系数据库

关系模型的数据结构

关系模型建立在集合代数基础上。

数据操纵

特点：操作对象和操作结果都是集合。

五种基本操作：选择、投影、并、差、笛卡尔积。

关系语言

关系演算

  • 高度非过程化的语言
  • 完备的
  • 三种语言在表达能力上完全等价

关系代数

以集合为基础，定义一组运算，由已知关系经过一系列运算，得到需要的查询结果的表达方式。

  • 关系代数是过程化的查询语言。
  • 并、交、差 要求参与运算的两个关系必须具有相同的结构。

概念

候选码是某一属性组的值能唯一标识一个元组，而其子集不能。

主码：若一个关系有多个候选码，则选其中一个为主码。

外码：一个表中一个（多个）属性，非自身的码，且该属性和另一个表的主码属性是同一属性，则这个（组）属性称为外码。

实体完整性：主码不能重复，不能为空。

参照完整性：外码不能为空（或均为空值），外码等于被参照表的主码值。

用户定义完整性：用户自定义的完整性约束条件。

三、SQL

SQL 功能 动词
数据查询 SELECT
数据定义 CREATE、DROP、ALTER
数据操纵 INSERT、UPDATE、DELETE
数据控制 GRANT、REVOKE
操作对象 操作方式
创建 删除 修改
模式 CREATE SCHEMA DROP SCHEMA
表 CREATE TABLE DROP TABLE ALTER TABLE
视图 CREATE VIEW DROP VIEW
索引 CREATE INDEX DROP INDEX ALTER INDEX

定义表格

PRIMARY KEY( xxx，xxx )：主码由两个或以上属性构成，必须作为表级完整性进行定义。

FOREIGN KEY( xxx，xxx ) REFERENCES TableName( xxx )

数据查询

DISTINCT：消除重复的行。

GROUP BY：对属性列值相等的值进行分组。

HAVING：对 GROUP BY 得到的值进行筛选。

ORDER BY 列名 DESC / ASC：查询结果降序或升序排列。

常用条件

比较符号：大于、小于等。

范围： BETWEEN AND、NOT BETWEEN AND。

确定集合： IN、NOT IN。

字符匹配： LIKE、NOT LIKE。

  • % 任意长度
  • _ 任意单字符
  • 加 \ 转义可以查上述字符。

多重条件：AND、OR、NOT。

聚集函数

COUNT(*)：统计元组个数。

SUM：列值和。

AVG：列值平均。

MAX：最大值。

MIN：最小值。

连接查询

等值连接和非等值连接。

自然连接：2张表公共属性等值连接，并将结果去掉重复列。

自身连接：自身笛卡尔积，再根据条件选择。

外连接：

  • 左外连接：保留左边，不匹配补空格。
  • 右外连接：保留右边，不匹配补空格。
  • 外连接：保留两边，不匹配补空格。

多表连接。

嵌套查询

分类：

不相关子查询：子查询的条件不依赖父查询。

相关子查询：子查询的查询条件依赖父查询。

ANY和ALL：必须同时使用比较运算符。

ANY：查询结果的某个值。

ALL：查询结果的所有值。

集合查询

INTERSECT 交、UNION 并、EXCEPT 差。

数据更新

插入数据 INSERT INTO VALUES。插入子查询结果。

修改 UPDATE SET WHERE。

删除 DELETE FROM WHERE。

视图

创建视图，删除视图。

查询视图，更新视图与基本表类似。

嵌入式 SQL

前缀 EXEC SQL

  1. 主变量

    • 主语言向SQL语句提供参数
    • 将SQL语句查询数据库的结果交主语言进一步处理

  2. SQL 通信区 ( SQLCA, SQL Communication Area )

    • 向主语言传递SQL语句的执行状态信息
    • 主语言能够据此控制程序流程

  3. 游标：　解决集合性操作语言与过程性操作语言的不匹配

    • 游标是SQL查询语句向宿主语言提供查询结果集的一段公共缓冲区

六、关系数据理论

外延和内涵

外延：关系模型的值，即关系模型中的数据。动态。

内涵：对关系、属性、域的定义和说明，即关系模型的型的定义。称为关系模式。静态。

规范化

规范化：定义关系模式应该符合的条件，确保关系模式不出现某些操作异常并减少数据冗余。

函数依赖

X 推出 Y。

平凡函数依赖：Y 是 X 的子集。

非平凡函数依赖：Y 不是 X 的子集。

完全函数依赖：X 的子集不能决定 Y。

部分函数依赖：X 的子集能决定 Y。

直接依赖：X 直接推出 Y。

传递函数依赖：X 推 Z，Z 推 Y。

多值依赖

多值依赖：X 推出多个 Y。

平凡多值依赖：上述条件下，X 和 Y构成全集。

非平凡多值依赖：上述条件下，X 和 Y不构成全集。

性质

对称性、传递性、若全集必能推出。

与函数依赖的区别

有效性、包含性。

概念

闭包（左边元素的集合）推出所有元素。

候选码由主属性构成，剩下非主属性。

函数依赖集的闭包：很难求。

属性的闭包：求到没有为止。

极小函数依赖集：右切、去本求包、左部最小化。

模式分解的无损连接性：交决定差。

候选码求解算法：分X和Y，X的闭包为全集则X是候选码，否则把Y的子集放进X中，若X为候选码则将子集从Y中去掉，直至子集为Y全集或Y为空。候选码可能多个。

范式

范式：1NF > 2NF > 3NF > BCNF。

1NF：每个元素不可分割。

2NF：上述情况下。不存在非主属性对码的部分依赖。

3NF：上述情况下。不存在非主属性之间的依赖关系。（不存在非主属性对码的传递依赖。）

BCNF：上述情况下。不存在主属性对候选码的部分或传递依赖。（决定因素都含有码。）

七、E-R 图

实体是矩形，属性是椭圆，联系是菱形。

根据题设画 E-R 图。

转换关系模式：

  • 每一个实体都是一个关系模式。
  • 1对1联系：将联系和另一个关系的主码加进另一个关系中。
  • 1对n联系：将联系的和 1 端关系的主码加进 n 端关系中。
  • m对n联系：联系自己成立关系模式，主码为两端关系的主码组。
