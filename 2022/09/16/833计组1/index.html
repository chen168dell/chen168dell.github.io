计算机系统概论

计算机的发展历史、摩尔定律

  • 20世纪70年代前，电子计算机包括电子模拟计算机和电子数字计算机。
  • 电子模拟计算机，由模拟电子器件构成，早已完成历史使命。
  • 电子数字计算机，是指能对离散逻辑符号表示的数据或信息进行自动处理的电子装置。

发展历史

  • 第0代：机械计算机
    • 如算盘、计算尺、手摇机械计算机
  • 第1代：电子管计算机（1945-1955）
    • 这一代计算机由电子管、电磁继电器等器件构成
    • 软件主要用机器语言编程
    • ENIAC：1943年研制
  • 第2代：晶体管计算机（1955-1965）
    • 晶体管、磁芯存储器等器件构成
    • 软件上采用监控程序对计算机进行管理，并开始使用高级语言
  • 第3代：集成电路计算机（1965-1980）
    • 小规模及中规模集成电路芯片、多层印刷电路板及磁芯存储器
    • 高级语言、分时操作系统
  • 第4代：超大规模集成电路计算机（1980-1990）
    • 大规模、超大规模集成电路，半导体存储器
    • 微型计算机、单片微型计算机
    • 著名操作系统：DOS、Windows、Unix
    • 这一时期应用最多最广的是个人计算机PC
  • 第5代：高性能智能计算机（1991-）
    • 超大规模、超高速集成电路
    • 多核处理器
    • 人性化、智能化

摩尔定律

  • 戈登摩尔 1965
  • 摩尔定律：集成电路芯片的集成度每18个月翻一番。
    • 摩尔定律并不是一个物理定律，而是一种预言。
  • ”若集成度提高100倍，则相对价格可以降低100倍、性能可提高100倍、可靠性也可以提高100倍。“
  • 摩尔定律的未来：新技术、新材料的出现，使芯片的集成度进一步提高成为可能。

计算机的基本组成

  • 硬件：有形的物理设备，计算机系统中实际物理装置的总称。
  • 软件：在硬件上运行的程序和相关数据及文档。
    • 软件系统包括：系统软件和应用软件。

软硬件的逻辑功能等价性：某一功能既可硬件实现又可软件实现，从用户角度看，它们再功能上是等价的。

硬件实现的功能要优于软件实现的功能。

硬件组成

一般将运算器和控制器集成到同一个芯片上，称为中央处理器（CPU）。

CPU 和存储器共同构成主机，除主机外的其他硬件装置（外存、I/O设备等）统称为外设。

CPU 和 主存之间通过总线相连，总线中有地址、数据、信号三组信号线。MAR 中的地址信息会直接送到地址线上，用于指向读写操作的主存存储单元。控制线中有读写信号线。

主存储器

存储器用来存放数据和程序。

存储器分为主存储器（内存）和辅助存储存储器（外存）。

CPU 能直接访问的存储器是主存储器，辅存帮助主存记忆更多信息。

辅存信息必须调入主存后才能被 CPU 访问。

主存储器的工作方式：按存储单元的地址进行存储。

  • 存储体存放二进制信息，数据在存储体内按地址存储。
  • MAR 存放访存地址，位数反映存储单元的位数。
  • MDR 暂存要从存储器中读写的信息，位数反映字长。
  • 时序控制逻辑用于产生存储器操作所需的各种时序信号。

1字节（Byte）=8 bit

1B = 1字节

1 b = 1bit

MAR 与 MDR 虽然是存储器的一部分，但在现代计算机中存在 CPU 中，高速缓存也存在 CPU 中。

运算器

运算器是计算机的执行部件，用于进行算术和逻辑运算。

  • ACC：累加器，用于存放操作数或运算结果。
  • MQ：乘商寄存器，在乘除运算时用于存放操作数或运算结果。
  • X：通用的操作数寄存器，用于存放操作数。
  • ALU：算术逻辑单元，通过内部复杂电路实现算术运算、逻辑运算。

控制器

控制器产生相应的控制信号，控制其他部分工作，以实现指令功能。

  • CU：控制单元，分析指令，给出控制信号。
  • IR：指令寄存器，存放当前执行的指令。
  • PC：程序计数器，存放下一条指令地址，有自动加一的功能。

完成一条指令：取指令、分析指令、执行指令。

取指：PC、IR

执行：CU

  • PC 与 主存的 MAR 之间有一条直接通路。
  • IR 用于存放当前执行的指令。
  • 指令中的操作码OP(IR)送至 CU，分析指令并发出各种微操作命令序列；地址码Ad(IR)送至 MAR，取操作数。

软件组成

  • 系统软件：保证计算机高效、正确运行的基础软件。
    • 操作系统
      • 管理计算机硬、软件资源，控制程序运行，改善人机交互并为应用软件提供支持。
      • 包括五大功能：处理器管理、存储管理、文件管理、设备管理及作业管理。
    • 语言处理程序
    • 通用程序
    • 各种服务支持软件
      • 各种调试程序、诊断程序、提示警告等
  • 应用软件：用户为解决某个应用邻域的各类问题而编制的程序。

三个级别的语言：

  • 机器语言：计算机唯一可以直接识别和执行的语言
  • 汇编语言：要经汇编程序翻译为机器语言
  • 高级语言：
    • 先经编译程序编译为汇编语言，再经汇编程序得到机器语言。
    • 直接由高级语言程序翻译称机器语言。

翻译程序：

  • 汇编程序（汇编器）：汇编语言到机器语言
  • 解释程序（解释器）：源程序语句顺序逐条翻译成机器指令并立即执行。
  • 编译程序（编译器）：高级语言到汇编语言或机器语言。

冯诺依曼计算机工作原理

冯诺依曼计算机结构及工作过程

计算机依据存储程序、程序控制的方式工作。

  • 输入设备：将程序和数据转换成机器能识别的形式。
  • 输出设备：将结果转换成人们熟悉的形式。
  • 存储器：存放数据和程序。
  • 运算器：算术运算和逻辑运算。
  • 控制器：指挥程序运行。

冯诺依曼计算机特点

  • 计算机由五大部件组成：运算器、存储器、控制器、输入设备、输出设备。
  • 指令和数据以同等地位存于存储器，可按地址寻访。
  • 指令和数据用二进制表示。
  • 指令由操作码和地址码组成。
  • 存储程序：将事先编制好的程序和原始数据送入主存后才能执行，一旦程序被启动执行就无须操作人员干预，计算机会自动逐条执行指令，直至程序执行结束。
  • 以运算器为中心：输入输出设备之间的数据传送通过运算器完成。

存储程序概念在冯诺依曼研究EDVAC机时提出，奠定了现代计算机的基本结构。

现代计算机以存储器为中心。

计算机系统的层次结构

层次结构图、各层的含义和界面定义

  • 第一级：微程序或硬件逻辑
    • 计算机设计者所要描述和设计的计算机。
    • 目前至将来第一级上硬件设计主要通过软件实现。
  • 第二级：机器语言
    • 系统中的指令必须是二进制编码。
    • 机器语言是针对具体硬件物理计算机的。
  • 第三级：操作系统
    • 设计计算机时应尽可能多地对操作系统给予支持。
    • 操作系统原则上是跨硬件平台的。
    • 就操作系统级来说，设计者既要了解硬件物理计算机也要了解软甲虚拟机。
  • 第四级：汇编语言
    • 汇编语言涉及的指令系统是针对特定处理器的，不同处理器的指令系统是不一样的。
  • 第五级：高级语言
    • 此级用户为高级语言程序设计者。
    • 跨平台的，无须知道更多计算机硬件细节。
  • 第六级：应用语言
    • 此级可设计更适应于应用的、更接近自然语言的编程语言，使使用者在虚拟机上工作，更少地依赖硬件系统。

下层是上层的基础，上层是下层的抽象。

计算机体系结构、计算机组成和计算机实现的定义

计算机系统结构决定计算机的总体属性，组成是体现这些属性的逻辑设计，而实现是用物理器件来实现逻辑设计。

  • 计算机系统结构
    • 从外部研究计算机系统，是使用者所看到的物理计算机的抽象，是编写出能够在计算机上正确运行程序所必须了解到的计算机属性。
    • 由（系统）程序员所看到的计算机系统的属性，即概念性结构及功能特性。
    • 包括：
      • 数据的表示形式
      • 寻址方式
      • 内部寄存器组
      • 指令集
      • 中断系统
      • 处理器工作状态及切换
      • 存储系统
      • 输入/输出结构
      • 信息保护及特权
      • 高性能设计等
    • 计算机系统结构界定了系统中硬件所实现的功能，其中最重要的就是指令集的实现。
  • 计算机组成
    • 从计算机内部研究计算机的构成。
    • 也称为计算机组织，是计算机系统的逻辑实现，包括最低层内部算法、数据流、控制流的逻辑实现。
    • 包括：
      • 数据通路的宽度
      • 专用部件的设置（如乘除法专用部件、浮点运算专用部件等）
      • 各功能部件的并行程度
      • 各操作的相容性与互斥性
      • 控制机构的组成方式
      • 缓冲和排队技术的应用
      • 预估、预判方法
      • 高可靠技术等
  • 计算机实现
    • 计算机组成的物理实现
    • 在计算机系统结构及计算机组成的基础上，利用具体的集成电路芯片、电子元器件、部件、插头、插座等，根据计算机组成的逻辑设计，实现物理计算机。

计算机的分类及性能描述

  • 早期的计算机分类方法
    • 人们根据计算机的字长、规模、价格等指标，将计算机分为微型机、小型机、中型机、大型机和巨型机。
  • 按用途分类
    • 通用计算机：可以满足许多用户的，按用途分为个人计算机 PC 和服务器。
    • 嵌入式计算机：以应用为目标，以计算机技术为基础，软硬件可裁剪，对各方面有严格要求的专用计算机系统。

Flynn 分类法

Flynn 分类法按照计算机在执行程序的过程中信息流的特征进行分类。

在程序执行过程中有三种信息流：

  • 指令流 IS：机器执行的指令序列，由存储器流入控制单元（CU）
  • 数据流 DS：指令流所使用的数据，包括输入数据、中间数据和结果。数据在处理单元（PU）中进行处理。
  • 控制流 CS：指令流进入控制单元，由控制单元产生一系列的控制流（信号），在控制流的控制流的控制下完成指令的功能。

Flynn 将计算机分为四类：

  • 单指令流单数据流计算机 SISD
    • 由单一控制单元、单一处理单元和单一主存储器组成。
    • 应用：传统的串行计算机
  • 单指令流多数据流计算机 SIMD
    • 由一个控制单元、多个处理单元和多个存储器模块组成。
    • 每次从存储器取一条指令到控制器，由控制器对指令进行译码，产生多个控制信号，并用相同的控制信号控制多个处理单元执行相同的操作，完成这条指令对多个数据的处理，最终实现一条指令所规定的功能。
    • 阵列处理机/向量处理机
  • 多指令流单数据流计算机 MISD
    • 由多个控制单元同时执行多条指令对同一数据进行处理。
    • 尚无实例。
  • 多指令流多数据流计算机 MIMD
    • 由多个控制单元、多个处理单元、多个存储模块构成，实际上是由多处理机用某种方式连在一起构成的计算机系统，通常称为多处理机系统。
    • 多核处理器、多处理机集群系统。

计算机的性能指标

  • 字长：计算机进行一次整数运算能处理的二进制数据的位数，通常与CPU的寄存器位数、加法器有关。
    • 字长一般等于内部寄存器的大小。
    • 字长越长数据表示的范围越大，计算机的精度越高。
    • 计算机字长常选为字节的整数倍。
  • 数据通路带宽：数据总线一次所能并行传送信息的位数。
    • 指外部数据总线的宽度，与CPU内部数据总线的宽度（内部寄存器的大小）有可能不同。
    • 数据通路：各个子系统通过数据总线连接形成的数据传送路径称为数据通路。
  • 主存容量：主存储器所能存储信息的最大容量。
    • 通常以字节衡量，也可用字数*字长来表示存储容量。
    • MAR 反映存储单元的个数，反映的是可寻址范围的最大值，不一定是实际存储的存储容量。
    • MDR反映每个存储单元的大小。

吞吐量、响应时间

  • 吞吐量：系统在单位时间内处理请求的数量。
    • 取决于信息能多块传入内存，CPU能多快取指令，数据能多快地从内存取出或存入，结果能多快地从内存传给一台外部设备。
    • 几乎每步都关系到主存，所以系统吞吐量主要取决于主存的存取周期。
  • 响应时间：从用户向计算机发送一个请求起，到系统对请求作出相应并获得所需结果为止，所用的时间。通常包括CPU时间与等待时间。
    • CPU 时间：运行一个程序所花费的时间。
    • 等待时间：用于磁盘访问、存储器访问、I/O操作、操作系统开销等时间。

CPU 时钟周期（或主频）、CPI、CPU 时间

  • CPU 时钟周期
    • 也称机器周期。在计算机中，为方便管理，常把一条指令的执行过程划分为若干阶段，每个阶段完成一项工作。
    • 如，取指令、存储器读、存储器写等，每一项工作称为一个基本操作（每个基本操作都由若干CPU最基本的动作组成）。
    • 完成一个基本操作所需要的时间称为机器周期。通常用内存中读取一个指令字的最短时间来规定CPU周期。
  • 主频：1s有几个时钟周期。
    • 也称CPU时钟频率。机器内部主时钟的频率，是衡量机器速度的重要参数。同型号的计算机，主频越高，完成指令的一个执行步骤所用的时间越短，执行指令的速度越快。
  • CPI 平均执行周期数：执行一条指令所需的时钟周期数。
    • 不同的指令，CPI 不同。
    • 相同的指令，CPI 也可能不同。
    • 程序或机器的 CPI 指该程序或该机器指令集中的所有指令执行所需的平均时钟周期数。
    • CPI一般和三种因素有关：
      • 程序的编写
      • ISA 指令集
      • 计算机的架构
  • CPU执行时间：运行一个程序所花费的时间。

CPU 性能（执行时间）取决于三个因素：主频、CPI和指令条数。

主频、CPI和指令条数是相互制约的。

MIPS、M/G/T/P/EFLOPS

  • MIPS：每秒执行多少百万条指令。
  • MFLOPS：每秒执行多少百万次浮点运算。

描述存储容量、文件大小时：K、M、G、T为2的幂次。

描述速度、频率时：K、M、G、T为10的幂次。
