存储器的分类，存储器的层次化结构，半导体随机存取器，主存储器 CPU 的连接，多模块存储器，高速缓冲存储器，虚拟存储器，磁盘存储器。

存储器的分类

按照不同方式进行分类的方法

  • 存储信息的介质
    • 半导体存储器：MOS型存储器、双极型存储器
    • 磁表面存储器：磁盘、磁带
    • 磁芯存储器
    • 光存储器：光盘
  • 在计算机中的用途
    • 主存储器
      • 主存、内存，存放计算机运行期间所需的程序和数据。
      • CPU 可对主存随机地进行访问，主存也可以和高速缓冲存储器（Cache）及辅存交换数据。
      • 主存特点：容量较少、存取速度较快、每位价格高。
    • 高速缓冲存储器：
      • Cache，位于主存和 CPU 之间，存放当前 CPU 经常使用的指令和数据，以便 CPU 高速访问它们。
      • Cache 的存取速度可与 CPU 的速度相匹配，但存储容量小、价格高。
      • 现代计算机通常将 Cache 制作在 CPU 中。
    • 控制存储器
    • 辅助存储器
      • 辅存，外存储器，存放当前暂时不用的程序和数据以及一些需要永久性保存的信息。
      • 辅存内容要调入主存后才能被 CPU 访问。
      • 辅存的特点：容量大、存取速度较慢、单位成本低。
      • 磁盘、光盘、闪盘、SD 存储卡、磁带。
  • 存放信息的易失（挥发）性
    • 易失性存储器：断电后存储信息即消失的存储器，RAM。
    • 非易失存储器：断电后信息仍保持的存储器，ROM、磁表面存储器、光存储器。
  • 存取方式
    • 破坏性读出：某存储单元所存储的信息被读出时，原存储信息被破坏。
      • 具有破坏性读出性能的存储器，每次读出操作后，必须紧接一个再生操作，以便恢复被破坏的信息。
    • 非破坏性读出：被读单元原存储信息不被破坏。
    • 随机读写：RAM
      • 任何一个存储单元都可以随机存取，存取时间与存储单元的物理地址无关。
      • 优点：读写方便、使用灵活。
      • 主要用作主存或高速缓冲存储器。
      • RAM 又分为静态 RAM 和动态 RAM。
    • 只读存储器：ROM
      • 只能随机读出而不能写入。
      • 信息一旦写入存储器就固定不变，即使断电，内容也不会丢失。
      • 通常用 ROM 存放固定不变的程序、常数和汉字字库等。
      • ROM 可与 RAM 共同作为主存的一部分，统一构成主存地址域。
      • ROM 派生出的存储器也包含可反复重写的类型，ROM 和 RAM 的存取方式均为随机存取。
      • 广义上的只读存储器已通过电可擦除等方式进行写入，其“只读”的概念没有保留，但仍保留了断电内容保留、随机读取的特性，但其写入速度比读取速度慢得多。
    • 顺序（串行）访问存储器
      • 对存储单元进行读写操作时，需要按其物理位置的先后顺序寻址。
      • 顺序存取存储器 SAM（磁带）
        • 内容只能按某种顺序存取，存取时间的长短与信息在存储体上的物理位置有关。
        • 特点：存取速度慢。
      • 直接存取存储器 DAM（磁盘、光盘）
        • 不能随机访问存取单元，也不完全顺序存取，而是介于两者之间。
        • 存取信息时通常先寻找整个存储器中的某个小区域（如磁盘上的磁道），再在小区域内顺序查找。

存储器容量、速度、可靠性

  • 存储容量：

    • 存储字数表示存储器的地址空间大小。
    • 字长表示一次存取操作的数据量。

  • 速度：

    • 存取时间 Ta：对存储器读/写一次的时间。
    • 存储周期 Tm：读写周期或访问周期，连续对存储器进行存（取）时，完成一次存（取）的时间。
    • 主存带宽 Bm：又称数据传输率，单位时间里存储器可以读写的字节数。
    • 通常存储周期大于存取时间。
      • 对任何一种存储器，在读写之后都需要恢复内部状态的复原时间。
      • 破坏性读出的存储器，存取周期往往比存取时间大得多。甚至 Tm=2Ta。

  • 可靠性

    • 可维修部件的可靠性：平均故障间隔时间
    • 不可维修部件的可靠性：平均无故障时间/平均故障前的时间

存储系统的层次化结构

存储器性能的基本要求：大容量、高速度、低价格。

存储器层次结构的主要思想：上一层的存储器作为低一层的存储器的高速缓存。

具体层次化形式

从 CPU 角度，Cache-主存层速度接近于 Cache，容量和位价却接近于主存。

从主存-辅存层分析，其速度接近于主存，容量和位价却接近于辅存。

现代计算机几乎都采用这种三级存储系统。

  • 主存和 Cache 之间的数据调动由硬件自动完成，对所有程序员均是透明的。
  • 主存和辅存之间的数据调动由硬件和操作系统共同完成，对应用程序员是透明的。

主存-辅存的不断发展中，逐渐形成了虚拟存储系统，系统中对程序员编程的地址范围与虚拟存储器的地址空间相对应。对具有虚拟存储器的计算机系统而言，编程时可用的地址空间远大于主存空间。

Cache-主存层和主存-辅存层中，上一层中的内容都只是下一层中的内容的一部分（副本）。

各种存储器的位置

半导体存储器：SRAM、DRAM、ROM(EPROM、EEPROM、Flash)存储器的工作原理、特点、适用场合

SRAM 和 DRAM

  • 存储元：存放二进制位的物理器件，存储器的最基本构件。
  • 地址码相同的多个存储元构成一个存储单元。
  • 若干存储单元的集合构成存储体。

SRAM 工作原理

  • 静态随机存取器 SRAM 的存储员用双稳态触发器（六晶体 MOS）记忆信息，非破坏性读出。
  • 优点：存取速度快。
  • 缺点：集成度低，功耗较大，价格昂贵。
  • 一般用于高速缓冲器。

DRAM 工作原理

  • 动态随机存储器 DRAM 利用存储元电路中栅极电容上的电荷存储信息。
  • DRAM 的基本存储元通常只使用一个晶体管，比 SRAM 密度高。
  • 相对于 SRAM，DRAM 易集成、价位低、容量大、功耗低。
  • 但 DRAM 的存取速度比 SRAM 慢，一般用于大容量主存系统。

DRAM 刷新

DRAM 电容上的电荷一般只能维持 1~2 ms，即使电源不断电，信息也会自动消失，故每隔一段时间必须刷新，通常取 2ms 称为刷新周期。

  • 集中刷新
    • 在一个刷新周期内，利用一段固定的时间依次对存储器所有行进行逐一再生。
    • 在此期间停止对存储器的读写操作，称为“死时间”，又称访存“死区”。
    • 优点：读写操作不受刷新工作影响。
    • 缺点：死区不能访问存储器。
  • 分散刷新
    • 把对每行的刷新分散到各个工作周期中。
    • 一个存储器的系统工作周期分为两部分：
      • 前半部分用于正常读、写或保持。
      • 后半部分用于刷新。
    • 优点：没有死区。
    • 缺点：加长系统存取周期，降低整机的速度。
  • 异步刷新
    • 前两种刷新方式的结合，既缩短死区又充分利用最大刷新时间间隔 2ms。
    • 将刷新周期除以行数得到两次刷新操作之间的时间间隔 t，利用逻辑电路每隔 t 时间产生一次刷新请求。
    • 避免使 CPU 连续等待过长的时间，减少了刷新次数，从根本上提高了整机的工作效率。

只读存储器 ROM

ROM 和 RAM 都支持随机存取。

SRAM 和 DRAM 均为易失性半导体存储器。

ROM 中一旦有了信息就不能轻易改变，即使掉电也不会丢失。

特点

  • 结构简单，位密度比可读写存储器的高。
  • 具有非易失性，可靠性高。

ROM 类型：根据制造工艺不同分类

  • 掩膜式只读存储器（MROM）
    • 厂家按客户需求在芯片生产过程中直接写入信息，之后任何人不可重写（只能读出）。
    • 优点：可靠性高，集成度高，价格便宜。
    • 缺点：灵活性差，生产周期长，只适合批量定制。
  • 一次可编程只读存储器（PROM）
    • 用户可用专门的 PROM 写入器写入信息，写一次之后就不可更改。
  • 可擦除只读存储器（EPROM）
    • 允许用户写入信息，之后用某种方法擦除数据，可进行多次重写。
    • UVEPROM：用紫外线照射 8~20 分钟，擦除所有信息。
    • EEPROM：可用“电擦除”方式，擦除特定的字。
  • 闪速存储器（Flash）
    • u 盘、SD 卡就是闪存。
    • 每个存储元只需要单个 MOS 管，位密度比 RAM 高。
    • 在 EEPROM 基础上发展而来，断电后也能保存信息，且可多次快速擦除重写。
    • 由于闪存需要先擦除后写入，因此闪存的“写”速度要比“读”速度更慢。
  • 固态硬盘（SSD）
    • 由控制单元+存储单元（Flash 芯片）构成，与闪速存储器的核心区别在于控制单元不一样，但存储介质都类似，可进行多次快速擦除重写。
    • SSD 速度快、功耗低、价格高，目前个人电脑上常用 SSD 取代传统的机械硬盘。

主存储器的构成方法

  • 主存储器由 DRAM 实现。
  • 靠处理器的那一层 Cache 由 SRAM 实现。
  • DRAM 和 SRAM 断电丢信息。
  • DRAM 每位价格低于 SRAM，速度慢于 SRAM。价格差异主要因为制造 SRAM 需要更多的硅。
  • ROM 非易失。

基本组成

  • 存储器核心部分：由记忆单元构成的存储矩阵（存储体）。
  • 记忆单元：具有两种稳态的能表示二进制0和1的物理器件。
  • 编址单位：具有相同地址的那些存储元件构成的一个单位，可以按字节编址，也可以按字编址。现代计算机通常采用字节编址方式。
  • 指令执行过程中需要访存时，CPU 首先把被访问单元的地址送到 MAR 中，通过地址线将主存地址主存的地址寄存器，以便地址译码器进行译码选中相应单元，同时 CPU 将读写信号通过控制线送到主存的读写控制电路。
    • 写操作：CPU 同时将要写的信息送到 MDR 中，在读写控制电路的控制下，经数据线将信号写入选中的单元。
    • 读操作：主存读出选中单元的内容送到数据线，然后送到 MDR 中。
  • 数据线的宽度与 MDR 宽度相同。
  • 地址线的宽度与 MAR 宽度相同。
  • 地址线的位数决定了主存地址空间的最大可寻址范围。
  • DRAM 芯片容量较大，地址位数较多，为了减少芯片的地址引脚数，通常采用地址引脚复用技术，行地址和列地址通过相同的引脚分先后两次输入，地址引脚数可减少一半。
  • 数据线和地址线共同反映存储容量大小。
  • 单个存储芯片容量有限，在字数或字长方面与实际存储器的要求有差距，常采用位扩展法、字扩展法、字位同时扩展法来扩展主存容量。

字扩展方式

增加存储器中字的数量而位数不变。

方式：将芯片的地址线、数据线、读写控制线相应并联，而由片选信号来区分各芯片的地址范围。通过片选信号 或采用译码器设计连接到相应的芯片。

位扩展方式

用多个存储器件对字长进行扩充，增加存储字长。

方式：将多个存储芯片的地址端、片选端和控制信号相应并联，数据端分别引出。片选信号要连到所有芯片。

位扩展+字扩展方式

各芯片连接地址线的方式相同，但连接数据线的方式不同，而且需要通过片选信号 或采用译码器设计连接到相应的芯片。

多体交叉（多模块）存储器：多体并行访问、多体交叉访问

多模块存储器是一种空间并行技术，利用多个结构完成相同的存储模块的并行工作来提高存储器的吞吐率。

CPU 的速度比存储器的快，若同时从存储器中取出 n 条指令，就可充分利用 CPU 资源，提高运行速度。多体交叉存储器就是基于这种思想提出的。

多体并行存储器由多体模块组成。每个模块都有相同的容量和存取速度，各模块都有独立的读写控制电路、地址寄存器和数据寄存器。既能并行工作又能交叉工作。

多体并行访问？？？

高位地址表示体号，低位地址为体内地址。

访问一个连续主存块时，总是先在一个模块内访问，等到该模块访问完才转到下一个模块访问，CPU 总是按顺序访问存储模块，各模块不能并行访问，因而不能提高存储器的吞吐率。

多体交叉访问

  • 低位地址表示体号，高位地址为体内地址。
  • 每个模块按“模 m”交叉编址。
  • 低位交叉方式下，总是把高位的体内地址送到低位体号确定的模块内进行译码。
  • 程序连续存放在相邻模块中，因此称采用此编址方式的存储器为交叉存储器。
  • 采用低位交叉编址后，可在不改变每个模块存取周期的前提下，采用流水线的方式并行存取，提高存储器的带宽。

高速缓冲器

研究 Cache-主存层。

程序局部性及 Cache 工作原理

程序局部性原理

程序访问的局部性原理包括时间局部性和空间局部性。

时间局部性指在最近的未来要用到的信息很可能是现在正在使用的信息，因为程序中存在循环。

空间局部性指在最近的未来要用到的信息很可能与现在正在使用的信息在存储空间上是邻近的，因为指令通常是顺序存放、顺序执行的，数据一般也是以向量、数组等形式簇聚地存储在一起的。

高速缓冲技术就是利用局部性原理，把程序中正在使用的部分数据存放在一个高速的、容量较小的 Cache 中，使 CPU 的访存操作大多数针对 Cache 进行，从而提高程序的执行速度。

Cache 基本工作原理

Cache 位于存储器层次结构的顶层，通常由 SRAM 构成。

为便于 Cache 和主存交换信息，Cache 和主存都被划分为相等的块，Cache 块又称 Cache 行，每块由若干字节组成，块的长度称块长。

Cache 的容量远小于主存的容量，所以 Cache 中的块数要远小于主存中的块数，仅保存主存中最活跃的若干块的副本。因此 Cache 按照某种策略，预测 CPU 在未来一段时间内欲访问的数据，将其装入 Cache。

当 CPU 发出读请求时，若访存地址在 Cache 命中，就将此地址转换成 Cache 地址，直接对 Cache 进行读操作，与主存无关；若 Cache 不命中，则仍需访问主存，并把此字所在的块一次性地从主存调入 Cache。若此时 Cache 已满，则需根据某种替换算法，用这个块替换 Cache 原来的某块信息。整个过程全部硬件实现。

CPU 和 Cache 之间的数据交换以字为单位，而 Cache 和主存之间的数据交换以 Cache 块为单位。

某些计算机采用同时访问 Cache 和主存的方式，若 Cache 命中则主存访问终止；否则访问主存并替换 Cache。

当 CPU 发出写请求时，若 Cache 命中，有可能会遇到 Cache 与主存中的内容不一致的问题，需要按一定的写策略处理，常见的处理方法有全写法和回写法。

CPU 欲访问的信息已在 Cache 中的比率称为 Cache 命中率。一个程序执行期间，Cache 的总命中次数为 Nc，访问主存的总次数为 Nm，则命中率 H 为 。设 tc 为命中时的 Cache 访问时间，tm 为未命中时的访问时间，则 Cache-主存系统的平均访问时间 Ta 为。

根据 Cache 的读、写流程，实现 Cache 时需要解决以下问题：

  • 数据查找：快速判断数据是否在 Cache 中。
  • 地址映射：主存块如何存放在 Cache 中，如何将主存地址转换为 Cache 地址。
  • 替换策略：Cache 满后，使用何种策略对 Cache 块进行替换或淘汰。
  • 写入策略：如何既保证主存块和 Cache 块的数据一致性，又尽量提升效率。

Cache 与主存地址映射与变换方式：全相联、直接映射、组相联

主存地址划分

Cache 行中的信息是主存中某个块的副本，地址映射是指把主存地址空间映射到 Cache 地址空间，即把存放在主存中的信息按照某种规则装入 Cache。

Cache 要为每块加一个标记，指明它是主存中哪一块的副本，标记的内容相当于主存中块的编号。为了说明 Cache 行中的信息是否有效，每个 Cache 行需要一个有效位。

Cache 行结构：标记项（标记位、有效位、一致性维护位、替换算法控制位）+数据

全相联：Tag（标记、区号）、Index（索引、块号、行号）、块内地址

主存中的每一块可以装入 Cache 中的任何位置，每行的标记用于指出该行取自主存的哪一块。故 CPU 访存时需要与所有 Cache 行的标记进行比较。

也就是说主存地址的高位为标记，低位为块内地址。

  • 优点：较灵活、Cache 块的冲突概率低，空间利用率高、命中率高。
  • 缺点：标记的比较速度较慢，实现成本较高。通常采用昂贵的按内容寻址的相联存储器进行地址映射。

访存过程就是根据主存地址的标记找到对应的 Cache 行，然后根据主存地址的块内地址在对应的 Cache 行中存取信息。

直接映射：Tag（标记、区号、组群号）、Index（索引、组号）、块内地址

主存中的每一块只能装入 Cache 中的唯一位置。

若这个位置已有内容，则产生块冲突，原来的块将无条件地被替换出去（无须使用替换算法）。

直接映射实现简单，但不够灵活，即使 Cache 的其他许多地址空着也不能占用，这使得直接映射的块冲突概率最高，空间利用率最低。

。

主存块号的低 c 位正好是它要装入的 Cache 行号。给每个 Cache 设置一个长为 t=m-c的标记（主存块号的高t位）,当主存某块调入 Cache 后，就将其块号的高 t 位设置在对应 Cache 行的标记中。

也就是说主存地址的高 t 位为标记（也称为主存区号），中间 c 位为要装入的 Cache 行号，末尾表示块内地址。

CPU 访存过程

  • CPU 首先根据访存地址（主存地址）中间的 c 位，找到对应的 Cache 行。
  • 将对应的 Cache 行中的标记和主存地址的高 t 位标记进行比较。
    • 若相等，且有效位为1，则访问 Cache 命中。此时根据主存地址中低位的块内地址，在对应的 Cache 行中存取信息。
    • 若标记不相等或有效位为0，则不命中。此时 CPU 从主存中读出该地址所在的一块信息送到对应的 Cache 行中，将有效位置1，并将标记设置为地址中的高 t 位，同时将该地址中（主存中）的内容送 CPU。

组相联：Tag（标记、区号、组群号）、Index（索引、组号）、块内地址

将 Cache 分成 Q 个大小相等的组，每个主存块可以装入固定组中的任意一行。即组间采用直接映射，组内采用全相联映射。

假设每个组有 r 个 Cache 行，则称之为 r 路组相联。

。

也就是说主存地址的高 t 位为标记，中间几位（位数由组数决定）为 Cache 组号，末尾几位为块内地址。

路数越大，即每组 Cache 行的数量越大，发生块冲突的概率越低，但相联比较电路也越复杂。

选定适当的路数，可以使组相联映射的成本接近直接映射，而性能上仍接近全相联映射。

CPU 访存过程

  • CPU 首先根据访存地址（主存地址）中间的组号找到对应的 Cache 组。
  • 将对应的 Cache 组中每个标记与主存地址的高位标记进行比较。
    • 若有一个相等且标记位为1，则访问 Cache 命中。此时根据主存地址中的块内地址，在对应的 Cache 行中存取信息。
    • 若都不相等或虽相等但标记位为0，则不命中。此时 CPU 从主存中读出该地址所在的一块喜喜送到对应 Cache 组的 任意一个空闲行 中，将有效位置1，并设置标记，同时将地址中（主存）的内容送CPU。

每个 Cache 行对应一个标记项，而在组相联中，将每组的标记项排成一行，将各组从上到下排列，成为一个二维的标记阵列（直接映射一行就是一组）。查找 Cache 时就是查找标记阵列的标记项是否符合要求。

三种映射的比较

三种映射方式中，直接映射的每个主存块只能映射到 Cache 中的某一固定行；全相联映射可以映射到所有 Cache 行；N 路组相联映射可以映射到 N 行。

当 Cache 大小、主存块大小一定时：

  • 直接映射的命中率最低，全相联映射的命中最高。
  • 直接映射的判断开销最小，所需时间最短；全相联映射的判断开销最大，所需时间最长。
  • 直接映射标记位所占的额外空间开销最少，全相联映射标记位所占的额外空间开销最大。

Cache 替换算法

直接映射时无须考虑替换算法。

常用替换算法：随机算法、先进先出算法、近期最少使用算法、最不经常使用算法。

  • 随机算法 RAND
    • 随机地确定替换的 Cache 块。
    • 优点：实现简单。
    • 缺点：未依据程序访问的局部性原理，可能命中率较低。
  • 先进先出算法 FIFO
    • 选择最早调入的行进行替换。
    • 优点：较易实现。
    • 缺点：未依据程序访问的局部性原理，因为最早进入的主存块也可能是目前经常要用的。
  • 近期最少使用算法 LRU
    • 依据程序访问的局部性原理，选择近期内长久未访问过的 Cache 行作为替换的行。
    • 平均命中率比 FIFO 高，是堆栈类算法。
      • 对堆栈类算法的个人理解：将整个 Cache 行空间视为一个栈，用到的行就往栈底压，其下的行自然就上浮了，若栈满了上浮时就自然会有个行溢出，就将其淘汰。
    • 算法：LRU 对每个 Cache 行设置一个计数器，计数值记录主存块的使用情况，根据计数值淘汰某个块。
      • 命中行计数值清零，比其低的计数器加1，其余不变。
      • 未命中且还有空行时，新装入的行计数器置0，其余加1。
      • 未命中且无空行时，计数值全1（二进制）的行淘汰，转上一条。
    • 缺点：当集中访问的存储区超过 Cache 组的大小时，命中率可能变得很低。比如 访问序列1234512345，而 Cache 只能存 4 行，命中率为 0，这种现象称为抖动。
  • 最不经常使用算法 LFU
    • 将一段时间内被访问次数最少的行换出。
    • 算法：每行设置一个计数器，被访问的行计数器加1，需要替换时比较各特定行的计数值，将计数值最小的行换出。
  • 最佳替换算法 OPT
    • 理论上的算法，选择淘汰的页面是以后永不使用的，或不需被访问时间最长的页面。

Cache 写策略 （Cache 一致性策略）

Cache 中的内容是主存块副本，当对 Cache 中的内容进行更新时，就需选用写操作策略使 Cache 内容与主存内容保持一致。

全写法

当 CPU 对 Cache 写命中时，把数据同时写入 Cache 和主存。当某一块需要替换时，不必将这一块写回主存，用新调入的块直接覆盖即可。

  • 优点：实现简单，能随时保持主存数据的正确性。
  • 缺点：增加了访存次数，降低了 Cache 的效率。

写缓冲

  • 为减少全写法直接写入主存的时间损耗，在 Cache 和 主存之间加一个写缓冲。CPU 同时写数据到 Cache 和写缓冲中，写缓冲再控制将内容写入主存。
  • 写缓冲是一个 FIFO 队列，可以解决速度不匹配的问题。
  • 但若出现频繁写时，会使写缓冲饱和溢出。

回写法

当 CPU 对 Cache 写命中时，只把数据写入 Cache，而不立即写入主存，只有当此块被换出时才写入主存。

  • 优点：减少访存次数。
  • 缺点：存在不一致性的隐患。

为减少写回主存的开销，每个 Cache 行设置一个修改位（脏位）：

  • 若脏位为1，说明被修改过，替换时要写回主存。
  • 若脏位为0，说明没修改过，替换时无须写回主存。

写分配法

Cache 写不命中时，加载主存的块到 Cache 中，然后更新这个 Cache 块。转为写命中的情况。

试图利用程序的空间局部性，但缺点是每次不命中都需要从主存中读取一块。

非写分配法

只写入主存，不进行调块。

多级 Cache 结构

随着新技术的发展（如指令预取），需要将指令 Cache 和 数据 Cache 分开设计，这就有了分离的 Cache 结构。

统一的 Cache 优点在于设计和实现相对简单，但执行部件存取数据时，指令预取部件要从同一 Cache 读指令，会引发冲突。

分离 Cache 结构可以解决这个问题，分离的指令和数据 Cache 还可以充分利用指令和数据的不同局部性来优化性能。

现代计算机的 Cache 通常设立多级 Cache 结构，离 CPU 越远访问速度越慢，容量越大。指令 Cache 与 数据 Cache 分离一般在 L1，此时通常写分配法和回写法合用。

虚拟存储器

研究主存-辅存层。

主存和辅存共同构成了虚拟存储器，二者在硬件和系统软件的共同管理下工作。

对于应用程序而言，虚拟存储器是透明的。

虚拟存储器具有主存的速度和辅存的容量。

虚拟存储器的基本概念

虚拟存储器将主存或辅存的地址空间统一编址，形成一个庞大的地址空间，在这个空间内，用户可以自由编程而不必在乎实际的主存容量和程序在主存中实际的存放位置。

  • 用户编程允许涉及的地址称为虚地址或逻辑地址，虚地址对应的存储空间称为虚拟空间或程序空间。
  • 实际的主存单元地址称为实地址或物理地址，实地址对应的是主存地址空间，也称实地址空间。
  • 虚地址比实地址大很多。

实地址=主存页号+页内字地址。

虚地址=虚存页号+页内字地址。

辅存地址=磁盘号+盘面号+磁道号+扇区号。

  • CPU 使用虚地址时，由辅助硬件找出虚地址和实地址之间的对应关系，并判断这个虚地址对应的存储单元内容是否已经装入主存。
    • 若已装入主存，则通过地址变换，CPU 可以直接访问主存指示的实际单元。
    • 若不在主存中，则把包含这个字的一页或一段调入主存后再由 CPU 访问。若主存已满，则采用替换算法置换主存中的交换块（即页面）

虚存采用和 Cache 类似的技术，将辅存中经常访问的数据副本存放到主存中。但缺页（或段）而访问辅存的代价很大，提高命中率是关键，因此虚拟存储机制采用全相联映射，每个虚页面可以存放到对应主存区域的任何一个空闲页位置。此外，在进行写操作时不能每次都写回磁盘，所以处理一致性问题时采用回写法。

段式、页式、段页式虚拟存储器

页式存储器

以页为基本单位。

虚拟空间和主存空间都机械等分成相同大小的页。

主存的页称为实页（页框），虚存的页称为虚页。

程序的起点必须处在某一页面的起点上。

页表

虚地址到物理地址的转换由页表实现。页表是一张存放在主存中的虚页号和实页号对照表，它记录程序的虚页调入主存时被安排在主存中的位置。页表一般长久地保存在内存中。

  • 有效位（装入位）：表示对应页面是否在主存。
    • 1，该虚拟页已从外存调入主存，此时页表项存放该页的物理页号。
    • 0，没有调入主存，此时页表项可存放该页磁盘地址。
  • 脏位（修改位）：表示页面是否被修改过。
    • 虚存机制中采用回写策略，利用脏位可以判断替换时是否需要写回磁盘。
  • 引用位（使用位）：配合替换策略进行设置。（比如 LRU 或 LFU 的计数器）

地址变换方法

  • CPU 执行指令时，先将虚拟地址转换为主存物理地址。
  • 页表基址寄存器存放进程的页表首地址，然后根据虚拟地址高位部分的虚拟页号找到对应的页表项。
    • 若有效位 1，取出物理页号，和虚拟地址的页内字地址拼接，形成实际的物理地址。
    • 若有效位 0，说明缺页，需要操作系统进行缺页处理。

页式虚拟存储器

  • 优点
    • 页面长度固定，页表简单，调入方便。
    • 主存利用率比较高。
    • 地址变换速度较快。
    • 对磁盘管理比较容易。
  • 缺点
    • 程序不可能正好是页面的整数倍，最后一页的零头将无法利用而造成浪费。
    • 页不是逻辑上独立的实体，所以处理、保护和共享都不如段式虚拟存储器方便。

段式存储器

存储器中的段按程序的逻辑结构划分，各个段的长度因程序而异。

虚拟地址分为：段号和段内地址。

虚拟地址到实地址之间的变换由段表实现。

段表

  • 段表是程序的逻辑段与主存中存放位置的对照表。
  • 段表的每行记录与某个段对应的段号、装入位、段起点和段长等信息。
  • 由于段的长度可变，所以段表中要给出各段的起始地址和段长。

地址变换方法

  • CPU 根据虚拟地址访存时，首先根据段号与段表基址拼接成对应的段表行，根据该段表行的装入位判断该段是否已调入主存。
    • 若调入主存，从段表中读出该段在主存中的起始地址，与段内地址（段内偏移）相加，得到对应的主存地址。

段式虚拟存储器

  • 优点
    • 段分界与程序自然分界相对应，具有逻辑独立性。模块化性能好。
    • 易于编译、管理、修改和保护。
    • 便于多道程序的共享。
    • 程序的动态链接和调度比较容易。
  • 缺点
    • 段长度可变，分配空间不便，容易在段间留下碎片，不好利用，造成浪费。
    • 地址变换所花费时间长，两次加法。
    • 段映像表庞大，地址、段长字段太长。
    • 对辅存的管理比较困难。（主存和磁盘之间交换信息的单位：页）

段页式存储器

程序按照逻辑结构分段，每段再划分为固定大小的页，主存空间也划分为大小相等的页，程序对主存的调入、调出仍以页为基本传送单位。

每个程序对应一个段表，每段程序对应一个页表，段的长度必须是页长的整数倍，段的起点必须是某一页的起点。

虚地址分为段号、段内页号、页内地址三部分。

  • CPU 根据虚地址访存时，首先根据段号得到段表地址。
  • 然后从段表中取出该段的页表起始地址，与虚拟地址段内页号合成，得到页表地址。
  • 从页表中取出实页号，与页内地址拼接形成主存实地址。

段页式存储器

  • 优点
    • 兼具页式和段式存储器的优点，可以按段实现共享和保护。
  • 缺点
    • 在地址变换过程中需要两次查表，系统开销较大。

TLB（快表、地址转换缓存、转换后援缓冲器）

访存时先访问一次主存取查页表，再访问主存才能取得数据，若缺页还要页面替换、页面修改等，因此采用虚拟存储机制后，访问主存的次数更多了。

依据程序的局部性原理，把在一段时间内总是经常访问的页表项存放在高速缓冲器组成的快表 TLB 中，可明显提升效率。

  • 把放在主存中的页表称为慢表 Page。
  • 在地址转换时，首先查找快表，若命中，则无须访问主存中的慢表。
  • 快表通常采用全相联或组相联方式。（一级全相联，二级组相联）

每个 TLB 项由页表表项内容加上一个 TLB 标记字段组成，TLB 标记用来表示该表项取自慢表中的哪个虚页号对应的页表项。

  • 在全相联方式下，TLB 标记内容就是慢表的页表项对应的虚页号。
  • 在组相联方式下，TLB 标记内容对应慢表虚页号的高位部分，虚页号的低位部分表示在 TLB 的哪个组。（参考组相联映射）

硬盘存储器

  • 硬盘存储器的组成：磁盘驱动器、磁盘控制器、盘片
    • 磁盘驱动器：核心部件是磁头组件和盘片组件。
      • 温彻斯特盘是一种可移动磁头、固定盘片的硬盘存储器。
    • 磁盘控制器：硬盘存储器和主机的接口，主流的标准有 IDE、SCSI、SATA 等。
  • 存储区域：一块硬盘含有若干记录面，每个记录面划分若干磁道，每条磁道又划分若干扇区，扇区（块）是磁盘读写的最小单位，即磁盘按块存取。
    • 磁头数
      • 即记录面数，表示硬盘共有多少个磁头。
      • 磁头用于读写盘面上记录面的信息，一个记录面对应一个磁头。
    • 柱面数
      • 表示硬盘每面盘片上有多少条磁道。
      • 在一个盘组，不同记录面的相同编号（位置）的诸磁道构成一个圆柱面。
    • 扇区数
      • 表示每条磁道上有多少个扇区。
  • 磁表面存储设备的特定
    • 优点
      • 存储容量大，位价格低。
      • 记录介质可重复使用。
      • 记录信息可以长期保存，可脱机存档。
      • 非破坏性读出。
    • 缺点
      • 存取速度较慢。
      • 机械结构复杂。
      • 对工作环境要求高。

磁记录原理

  • 原理：磁头和磁性记录介质相对运动时，通过电磁转换完成读写操作。
  • 编码方法：按某种方案（规律），把一连串二进制信息变换成存储介质磁层中一个磁化翻转状态的序列，并使读写控制电路容易、可靠地实现转换。
  • 磁记录方式：通常采用调频制 FM 和改进型调频制 MFM 的记录方式。

磁盘的基本参数及计算

  • 记录密度：盘面单位面积上记录的二进制信息量，通常以道密度、位密度和面密度表示。
    • 道密度：沿磁盘半径方向单位长度上的磁道数。
    • 位密度：磁道单位长度上能记录的二进制代码位数。
    • 面密度：道密度和位密度的乘积。
  • 存储容量
    • 非格式化容量：
      • 磁记录表面可利用的磁化单元总数。
    • 格式化容量
      • 按照某种特定的记录格式所能存储信息的总量。
      • 格式化后的容量比非格式化容量要小。
  • 平均访问时间：平均寻道时间+平均旋转延迟时间+平均传输时间
    • 寻道时间：磁头移动到目的磁道的时间。
    • 旋转延迟时间：磁头定位到要读写扇区的时间。一般选用磁盘旋转一周所用时间的一半作为平均旋转延迟时间。
    • 数据传输时间：传输数据所花费的时间。
  • 数据传输率：磁盘存储器在单位时间内向主机传送数据的字节数。

磁盘阵列（RAID）

RAID（独立冗余磁盘阵列）是指将多个独立的物理磁盘组成一个独立的逻辑盘，数据在多个物理盘上分割交叉存储、并行访问，具有更好的存储性能、可靠性和安全性。

在 RAID1~RAID5 几种方案中，无论何时有磁盘损坏，都可随时拔出受损的磁盘再插入好的磁盘，而数据不会损坏，提高了系统的可靠性。

  • RAID0：无冗余和无校验的磁盘阵列。

  • RAID1：镜像和磁盘阵列。

  • RAID2：采用纠错的海明码的磁盘阵列。

  • RAID3：位交叉奇偶校验的磁盘阵列。

  • RAID4：块交叉奇偶校验的磁盘阵列。

  • RAID5：无独立校验的奇偶校验磁盘阵列。

RAID0 把连续多个数据块交替地放在不同物理磁盘的扇区中，几个磁盘交叉并行读写，不仅扩大了存储容量，而且提高了磁盘数据的存取速度，但 RAID0 没有容错能力。

为了提高可靠性，RAID1 使用两个磁盘同时进行读写，互为备份，若一个磁盘出现故障，可以从另一个磁盘中读取数据。两个磁盘当一个磁盘使用相当于磁盘容量减少一半。

总之，

  • RAID 通过同时使用多个磁盘，提高了传输率。
  • 通过在多个磁盘上并行存取来大幅提高存储系统的数据吞吐量。
  • 通过镜像功能，提高安全可靠性。
  • 通过数据校验，提高容错能力。
