补码定点数的加/减运算

方法，应用

  • 补码加减法（设机器字长为n+1）

  • 特点：

    • 逢二进一。
    • 加法直接加，减法为加负数补码。
    • 符号位与数值位一起运算，符号结果也在运算中得出。
    • 最终运算结果的高位丢弃，保留 n+1 位，结果还是补码。

行波进位加法器

n 个全加器相连得到的 n 位加法器称为串行进位加法器，又称为行波进位加法器，每级进位直接依赖于前一级进位，即进位信号是逐级形成的。

实现两个 n 位二进制数逐位相加的功能。

由于位数有限，高位自动丢失，故实际是模的加法运算。

行波进位加法器的最长运算时间主要由进位信号的传递时间决定，位数越多延迟时间越长，而加法器本身的求和延迟只为次要因素。

加快进位产生和提高传递速度是关键。

先行进位加法器

令 ，。

  • 进位产生函数，本地进位。
  • 进位传递函数，进位传递条件。

实现上述逻辑表达式的电路称为线性进位部件或超前进位部件，简称 CLA 部件。

因为各进位并行产生，所以是一种并行加法器。

这种进位方式快速且与位数无关。

但随着加法器位数的增加， 的逻辑表达式会变得越来越长，电路结构会变得很复杂。因此当位数较多时，使用全先行进位不现实。

可以将 CLA 部件串接，组内先行进位，组间串行进位，也可组内和组间都并行进位。

通常采用两级或多级先行进位加法器。

溢出处理和判断

当且仅当两个同符数相加或异符数相减时才可能产生溢出。

  • 采用一位符号：由于减法在机器中用加法实现，故当参加操作的两个数符号相同且与结果不同，表示溢出。
  • 采用双符号：
    • 00 正无溢
    • 01 正溢
    • 10 负溢
    • 11 负无溢
  • 进位判决法：采用一位符号位根据数据位的进位判断
    • 最高数值位向符号位的进位与符号位的进位相同，则不溢出，否则溢出。
  • 根据运算结果的符号位和进位标志判别
    • 运算结果和进位标志符号相同，不溢出；否则，溢出。

在 CPU 中，进行定点算术运算是否发生溢出，通常是由 CPU 的硬件逻辑电路进行检测，一旦发生溢出就会：

  • 在 CPU 中的标志寄存器中建立溢出标志 OF。
  • 或产生溢出中断。

防止溢出：增加补码的二进制编码长度。

8421 BCD 数加法器

  • 定义
    • 压缩 BCD 数：四位二进制数表示一位BCD码，一个字节表示的两位BCD码，即一个字节存放两个十进制数，更节省存储空间。
    • 非压缩 BCD 数：一个字节存放一个一位十进制数，高4位的内容不做规定（也有部分书籍要求为0，二者均可），低4位二进制表示该位十进制数。
  • 加法运算
    • 运算中某位 BCD 数相加的结果大于9或有向更高位的进位，则结果加6。
    • 若不满足上述条件，则无需校正。

定点数的乘/除运算

乘法：原码一位乘法、补码一位乘法（Booth 法）、阵列乘法器

原码乘法运算

  • 符号位单独处理：被乘数符号与乘数符号异或。
  • 乘积的绝对值为被乘数的绝对值与乘数的绝对值之积。

  • 部分积：乘法过程中的中间结果，初始化全0.
  • 从乘数的最低位开始判断：
    • 若为1，部分积加上被乘数，逻辑右移。
    • 若为0，部分积加上0，逻辑右移。
  • 部分积取单符双符均可，建议双符。

补码乘法运算（Booth法）

  • 符号位参与运算，运算数均以补码表示。
  • 被乘数取双符参与运算，部分积取双符号位初值为全0，乘数取单符号位。
  • 乘数末位增设附加位（辅助位），初值为0。

恢复余数法

符号位单独处理，数值位取绝对值进行除法运算。

  • 老余数-|除数|=新余数
    • 新余数为正，上商1，余数逻辑左移。
    • 新余数为负，上商0，新余数+|除数|恢复老余数，余数逻辑左移。

加减交替法（原码）

  • 先用被除数减除数
    • 余正，上商1；余数和商左移，再减除数。
    • 余负，上商0；余数和商左移，再加除数。
  • 当第 n+1 步余数为负时，需要加上除数恢复余数。

  • 加/减 n+1 次，每次加减确定一位商。
  • 左移 n 次，最后一次加减完不移位。
  • 最终可能要多加一次恢复余数。

加减交替法（补码）

特点：符号位与数值位一起参与运算，商符自然形成。

  • 符号参与运算，除数与被除数均用补码表示，商和余数也用补码表示。
  • 若被除数与除数
    • 同号：减除数
    • 异号：加除数
  • 若余数与除数
    • 同号：上商1，左移，减除数
    • 异号：上商0，左移，加除数
  • 重复第三步 n 次。
  • 若对商的精度没有要求，采用”末位恒置1“法。

浮点数的加/减运算

阶码运算和尾数运算分开进行。

对阶

使两个操作数的小数点位置对齐，即使两个数阶码相等。

原则：小阶对大阶。小阶码+1，尾数右移1位，直到增大到与大阶码相同。

尾数右移时，舍弃掉有效位会产生误差影响精度。

尾数加减

对阶后的尾数按定点数加减运算规则运算。

尾数结果不一定规格化，要进一步规格化处理。

规格化：左规、右规

  • 右规：

    • 结果为 时要右规。
    • 尾数右移一位，阶码加1。
    • 阶码加一，必须判断是否上溢。若上溢可认为结果无穷。

  • 左规：

    • 结果为 时要左规。
    • 尾数左移一位，阶码减1。
    • 阶码减一，必须判断是否下溢。若下溢可认为结果为0.

PS：左规相当于乘2，右规相当于除以2，要右规时只需进行一次。

运算结果规格化、舍入

在对阶和尾数右规时，可能会对尾数右移，为保证运算精度，一般将低位移出的两位保留，参加中间过程的运算，最后将运算结果进行舍入，还原表示成 IEEE754。

  • 0舍1入：保留位最高位为0舍去，为1尾数末位加1，尾数可能会溢出，要右规。
  • 恒置1法：保留位不为0，则尾数末位置1.
  • 截断法：截取所需位数，丢弃后面所有位，这种舍入处理最简单。

逻辑与移位运算

  • 算术移位（有符号数）：移位过程中符号位不变。
    • 原码：左移右移均补0。
    • 反码：左移右移均补1。
    • 补码：左移补0，右移补符。
    • 带符号数，左移不溢出相当于乘2，右移不考虑末位尾数舍弃相当于除以2。
  • 逻辑移位：将操作数视为无符号数。左移右移均补0。
  • 循环移位
    • 大循环：带进位标志位 CF 的循环移位。
    • 小循环：不带进位标志位的循环移位。移出的也会入进位。
    • 循环：移出的数据又移入数据中。
    • 循环移位操作特别适合将数据的低字节数据和高字节数据互换。
