833计组：计算机系统中的数据表示

发表于 2022-09-17 更新于 2022-09-19 分类于 计算机组织与体系结构 阅读次数： Valine：

数制与编码，定点数据表示，浮点数据表示，检错与纠错码。

数制与编码

计算机采用二进制编码的原因：

  • 二进制只有两种状态，使用有两个稳定状态的物理器件就可以表示二进制数的每一位，制造成本较低。
  • 二进制位1和0正好与逻辑值”真“和”假“对应，便于实现逻辑运算和程序中的逻辑判断。
  • 二进制编码和运算规则简单，通过逻辑门电路能方便地实现算术运算。

进位计数法：

  • 每个数位用到的不同数码个数称为基数。
  • 每个数码表示的数值等于该数码本身乘以一个与它所在数位有关的常熟，这个常熟称为位权。
  • 一个进位数的数值大小为它的各位数码按权相加。

数制及相互转换

  • 二进制转八进制、十六进制
    • 整数部分：从小数点向左分组（3位/4位）
    • 小数部分：从小数点向右分组（3位/4位）
    • 分别用对应的八/十六进制数取代。
  • 八进制/十六进制转二进制
    • 每位改为3/4位二进制数
  • 任意进制数转换为十进制数
    • 按权展开相加：任意进制数的各位数码与它们的位权相乘再相加。
  • 十进制数转任意进制数：基数乘除法
    • 整数部分除基取余，先低后高。
    • 小数部分乘基取整，先高后低。

计算机中整数可以连续表示，但是小数是离散的。所以并不是每个十进制小数都可以准确地用二进制表示。但任意一个二进制小数都可以用十进制表示。

真值和机器数

  • 带 ”+“ 或 ”-“ 符号的数称为真值。真值是机器数所代表的实际值。
  • 计算机中通常将数的符号和数值部分一起编码，将数据的符号数字化，通常用0表示正，用1表示负。这种把符号数字化的数称为机器数。
  • 机器数的定点表示
    • 定点小数，纯小数，约定小数点位置在符号位之后，有效数值部分最高位之前。
    • 定点整数，纯整数，约定小数点位置在有效数值部分最低位之后。

BCD 码

  • 十进制有权码
    • 8421码：加6修正
    • 2421码：权值由高到低分别为2421，大于等于5的四位二进制数最高位为1
  • 十进制无权码
    • 余3码：在8421码的基础上加一个（0011）

字符与字符串

ASCII 字符编码

  • 7位，高位补0，在计算机中按8位
  • 33个控制字符
  • 0-9：30H加上相应数字值
  • 52个英文字母：
    • 大写字母：40H
    • 小写字母：60H
  • 32个特殊字符
  • 空格符号

汉字的表示

  • 汉字的输入编码
    • 拼音
    • 字形
    • 数字
  • 汉字的内码
    • GB2312-1980（国标） 6763个汉字+682个其它符号
    • GBK 21003个汉字+883个其它符号
    • GB18030-2000 27484个汉字+少数民族文字
  • 汉字的输出
    • 点阵 16*16点阵 32字节
    • 矢量 构成字的折线导数
    • 曲线 用曲线，记录曲线方程参数

定点数据表示

根据小数点的位置是否固定，在计算机中有两种数据格式：定点表示和浮点表示。

现代计算机中，通常用定点补码整数表示整数，用定点原码小数表示浮点数的尾数部分，用移码表示浮点数的阶码部分。

无符号数表示

  • 全部二进制位均为数值位而无符号位，默认数的符号为正。
  • 一般在全部是正数运算且不出现负值结果的场合下使用。
  • 可用无符号整数进行地址运算，或用它来表示指针。

有符号数表示

补码有明显优势

  • 0的补码表示唯一。
  • 补码运算规则简单，且符号位可以和数值位一起参加运算。
  • 补码多表示一个最小负数。

计算机中的带符号整数都用补码表示，故 n 位带符号整数的表示范围是 ~ 。

原码、反码、补码、移码：定义、性质，各种编码及真值之间的相互转换

  • 原码、补码、反码的符号位相同，正数的机器码相同。
  • 原码、反码的表示在数轴上对称，二者都有正零和负零。
  • 补码、移码的表示在数轴上不对称，零的表示唯一且比原码、反码多表示一个最小负数。
  • 整数的补码、移码的符号位相反，数值位相同。
  • 负数的反码、补码末位相差1。

原码

机器数的最高位表示数的符号，其余各位表示数的绝对值。

  • 纯小数的原码
    • 若字长为n+1，范围，原点对称。

  • 纯整数的原码
    • x 是真值，n 是整数位数
    • 若字长为 n+1 ，范围，原点对称。

  • 真值零的原码表示有正零和负零两种形式。
  • 原码的优缺点
    • 优点：
      • 与真值的对应关系简单、直观。
      • 与真值转换简单。
      • 用原码实现乘除运算比较简便。
    • 缺点：
      • 0的表示不唯一。
      • 原码加减运算比较复杂。

补码

补码表示法中的加减运算统一采用加法操作实现。

  • 有模运算：在实数范围内进行的运算。
  • 有模运算：在一定数值范围内进行的运算。
    • 减一个数=加上这个数的补数
    • 如：在时钟里 8-3=5，8+9=5

设模为 M，一个数 x 补码一般定义为：。

  • 纯小数的补码
    • 模2
    • 比原码多表示 -1 1.0000000
  • 纯整数的补码
    • 模
    • 比原码多表示 10000000
  • 零的补码表示是唯一的。

  • 补码与真值的转换
    • 正数补码的转换方式与原码相同。
    • 负数，符号位取1，其余各位取反，末位加1。
  • 求补运算
    • 按位取反，末位加一。
    • 自低位向高位搜索，第一个1及其以右各位0保持不变，以左各高位按位取反。

  • 补码的符号位的扩展 n->2n
    • 定点小数：在其低位填充适当位数的0。
    • 定点整数：符号位扩展，高位补n个符。
  • 补码的移位运算
    • 除2 算数右移：按位右移，高位补符。
    • 乘2 算数左移：按位左移，低位补零。
  • 变形补码：采用双符号位，左符是真正的符号位，右符用来判断溢出。
    • 左右相同，不发生溢出。
    • 左右相异，发生溢出。

反码

正数反码与原码、补码表示相同。

负数反码为对应补码末位减1。

反码的缺点：

  • 0 的表示不唯一
  • 范围比补码少一个最小负数。

反码在计算机中很少使用，通常用作数码变换的中间表示形式。

移码

移码常用来表示浮点数的阶码，只能表示整数。

移码就是在真值 x 上加一个常数（偏置值），这个常数取，相当于 x 在数轴上正方向偏移了若干单位。

特点：

  • 移码中零的表示唯一。
  • 一个真值的移码和补码仅差一个符号位。
  • 移码全0表示最小值，全1表示最大值。
  • 移码保持了数据原有的大小顺序。

移码的加减运算：

  • 对机器数直接求和差。
  • 对结果进行修正：结果的符号位取反。

浮点数据表示

以适当的形式将比例因子表示在数据中，让小数点的位置根据需要而浮动。

在位数有限的情况下，扩大数的表示范围，保持了数的有效精度。

格式定义

浮点数的表示形式：

  • S 取值0或1，决定浮点数的符号。
  • M 是一个二进制定点小数，称为尾数，用定点原码小数表示。
  • E 是一个二进制定点整数，称为阶码或指数，用移码表示。
  • R 是基数（隐含），约定为2、4、16等。

浮点数由数符、尾数和阶码三部分组成。

  • 阶码值反映浮点数的小数点的实际位置。
  • 阶码位数反映浮点数的表示范围。
  • 尾数位数反映浮点数的精度。

规格化

尾数的位数决定浮点数的有效数位，有效数位越多，数据的精度越高。

规格化操作通过调整一个非规格化的浮点数的尾数和阶码的大小，使非零的浮点数在尾数的最高数位上保证是一个有效值。

  • 左规：尾数左移一位，阶码减一（基数为2时），直至尾数变成规格化形式。
  • 右规：尾数右移一位，阶码加1（基数为2时）。

  • 当浮点数尾数的基数为4时，原码规格化数的尾数最高两位不全为0。

表示数的范围

原码关于原点对称，故浮点数范围也关于原点对称。

浮点数分布是不均匀的，越靠近0越密集。

  • 运算结果大于最大正数时称为正上溢，小于绝对值最大负数时称为负上溢，统称上溢。
  • 上溢发生时，计算机必须中断运算操作，进行溢出处理。
  • 运算结果在0至最小正数之间时称为正下溢，在0至绝对值最小负数之间时称为负下溢，统称下溢。
  • 下溢发生时，浮点数值趋于零，计算机将其当作机器零处理。
  • -129的原因：阶码移码表示，多一个最小负数，可以表示到-128，而尾数最小正数为1/2,综合起来就是-129了。

IEEE754 标准

  • 便于程序从一类处理器移植到另一类处理器上。
  • 研制更为复杂的数值运算程序。

基数隐含为2。

IEEE754 标准的浮点数（除临时浮点数外），是尾数采取隐藏位策略的原码表示，且阶码用移码表示的浮点数。

隐藏位：对规格化的二进制浮点数，数值的最高位总是”1“，为了使尾数多表示一位有效位，将这个”1“隐藏。

  • float 偏置值为127
  • double 偏置值为1023
  • 阶码=真值+偏置。
  • 上式中 E(float)为1~254 表示-126~127，E(double)为1~2046 表示-1022~1023。
  • IEEE754格式浮点数，阶码为全0或1时，有特别解释。
    • 全0阶码全0尾数：+0/-0
    • 全1阶码全0尾数：正无穷/负无穷
      • 引入无穷大的目的是，在计算过程出现异常的情况下使得程序能继续进行下去。

检错与纠错码

奇偶校验码

设 是一个 n 位字，则：

  • 奇校验位定义为： 当 X 中包含奇数个1时为1。
  • 偶校验位定义为： 当X 中包含偶数个1时为0。

缺点：

  • 只能检测每个字中产生的奇数个错误。
  • 不具备纠错能力

优点：

  • 开销小。
  • 通常用于1字节长的数据编码发生错误时，1位出错的概率较大，两位以上同时出错的概率极小。

常用于：

  • 存储器读写校验
    • 内存的存取过程中，发生一位错的概率最大。
    • 电路简单、速度高、易于实现。
  • 按字节传输过程中的数据校验。

水平垂直校验码（二维）：

  • 可发现并纠正数据块中的一位错误。
  • 可发现两位同时出错，但无法纠正错误。（少了一个坐标无法定位）

海明校验码

某些系统需要具备纠正合理数量错误的能力。

海明码：

  • 主要用于存储器数据的检验与纠正。
  • 采用奇偶校验的原理，错误检测和校正能力随着信息字加入奇偶校验位的数目线性增加。
  • 适用于最有可能发生随机错误的系统。
    • 每一位出错概率相同。
    • 每一位与其它位是否出错没有任何关联。
